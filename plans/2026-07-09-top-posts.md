# Top Posts — X/Threads composer on Top Clips + BLAST handoff

## Context

Extends Top Clips (plans/2026-07-09-top-clips.md, merged PR #4): the same
evidence-grounded pipeline, but the output is a text post. A transcript line
that matches a proven hook IS a tweet. Companion change in the blast repo
(plans/2026-07-09-threads-and-handoff.md) adds Threads + prefilled intents +
the handoff consumer; each side ships safely alone.

## What was built

- **→ POST** button on every Top Clips card opens an inline composer with two
  variants: X (≤280) and Threads (≤500), each with a live counter, COPY, a
  **prefilled compose intent** link (x.com/intent/post, threads.net/intent/post),
  and **SEND TO BLAST**.
- **Offline (no key): quote + attribution.** Verbatim line, trimmed on a word
  boundary with an ellipsis (the result is always a strict prefix of the
  original — the proven line is truncated, never rewritten), wrapped in curly
  quotes + a `{show}` attribution template (Settings → CHANNEL → Post
  attribution; default "— {show}"). Budget math guarantees ≤ limit including
  the attribution; a pathological template drops attribution rather than the line.
- **AI compose (key set):** one jsonMode call grounded in the line + its
  evidence (matched pattern/winner/reason) + channel fields; client-side limit
  enforcement; any failure keeps the offline drafts and toasts.
- **Handoff:** SEND TO BLAST writes `blast_handoff_v1` {caption, source,
  createdAt} and opens ../blast/ (same-origin). BLAST imports it only when no
  caption is in progress. Standalone deploys: the key is never read — no-op.
- Panel state survives refresh() (openPostKey/postDrafts module state); typing
  is surgical (no re-render, no blur); exit() closes the panel. SW cache
  v11 → v12. README documents Top Clips + Top Posts.

## Files touched

`topclips.js` (composers, panel, wiring, handoff), `app.js` (postAttribution
prefill/persist), `index.html` (Post attribution input), `style.css` (.post*
styles), `sw.js` (cache bump), `README.md`.

## Rollback

One squash commit; revert redeploys prior state. Additive settings key only;
drafts are session RAM. The handoff key goes unread if reverted.

## Verification

Headless Chromium, parent-dir server (recall/blast/Hooklabs same-origin):
offline X draft ≤280 and is a prefix-quote of the card line with srcTitle
attribution; Threads ≤500; intent hrefs decode to the textarea values; typing
updates count/href without closing the panel; stubbed AI compose replaces both
variants; SEND TO BLAST writes a valid payload and opens ../blast/; +BIN with
an open panel survives refresh; cross-app: BLAST (merged Threads build) imports
the caption on boot; zero console errors. Results recorded on execution.
