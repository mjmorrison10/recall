# Stack switcher: one-click switching between the four apps (RECALL side)

## Goal

Switching apps was painful: RECALL's only links to HOOKLAB/BLAST went to the
mjmorrisonusa promo pages, where you have to scroll to find the actual app
link. Add a slim switcher bar at the top of every app that jumps straight to
the live apps, and point the footer links at the live apps too.

## Steps (RECALL)

1. Vendor `stacknav.js` (byte-identical across all four apps, same pattern as
   `stackdata.js`). Classic script; on load it prepends a `nav.mjm-stacknav`
   bar as the first element of `<body>`: "THE STACK" label + four pills in each
   app's brand color. The current app is a solid non-link pill
   (`aria-current="page"`); the other three are same-tab links to the live
   GitHub Pages URLs. Detects the current app from `location.pathname`
   (checking `pulse` before `blast`, since PULSE lives at /blast/pulse/).
2. `index.html`: load `stacknav.js` after `stackdata.js`; footer `flinks` now
   point at the live apps (HOOKLAB, BLAST, + new PULSE) instead of the promo
   pages; footer tagline extended through PULSE.

## Files touched

- `stacknav.js` (new, vendored)
- `index.html`

## Rollback

Revert the branch; two files, no JS-app or storage changes.

## Verification

Same-origin headless Chromium harness across all four apps: bar present and
first in body, current app highlighted correctly per app, exactly 3 links each
pointing at the correct live URLs, no page errors. 24 assertions pass, plus
desktop/mobile screenshots eyeballed (pills wrap cleanly at 375px).
